# 주간업무 취합 시스템

팀장(팀별 현황)과 개인(개인별 계획)이 각자 웹 폼으로 입력하면, Google Sheets에 자동 저장되고
대시보드에서 실시간으로 취합/분석되며, 버튼 한 번으로 기존 "주간보고" 엑셀 양식으로 내보낼 수 있습니다.

## 구성

```
apps-script/
  Code.gs           # 백엔드 (Google Apps Script) - Sheets를 DB로 사용
  appsscript.json    # 매니페스트 (권한 범위)
team-form.html        # 팀장용 입력 폼
individual-form.html  # 개인용 입력 폼
dashboard.html         # 부실장님용 취합 대시보드
```

## 1. Google Sheets + Apps Script 준비

1. 새 Google 스프레드시트를 만듭니다. (시트 탭은 자동 생성되니 비워둔 채로 둡니다)
2. 확장 프로그램 > Apps Script 를 열고, `apps-script/Code.gs` 내용을 붙여넣습니다.
3. 왼쪽 프로젝트 설정에서 `appsscript.json`이 보이도록 "매니페스트 파일 표시"를 켠 뒤,
   `apps-script/appsscript.json` 내용으로 덮어씁니다.
4. 배포 > 새 배포 > 유형: **웹 앱**
   - 실행 계정: 나
   - 액세스 권한: 조직 내 계정만 접근하게 하려면 "조직 내 사용자"로, 외부 접근도 허용하려면 "전체 허용"으로 설정
5. 배포 후 나오는 웹 앱 URL(`https://script.google.com/macros/s/xxxx/exec`)을 복사해 둡니다.

## 2. HTML 파일에 API 주소 연결

`team-form.html`, `individual-form.html`, `dashboard.html` 세 파일 상단의

```js
const API_URL = 'https://script.google.com/macros/s/여기에_배포_ID/exec';
```

를 1번에서 복사한 실제 웹 앱 URL로 각각 교체합니다.

## 3. GitHub Pages 배포

기존에 쓰시던 방식과 동일하게, 이 세 HTML 파일을 GitHub 저장소에 올리고 GitHub Pages를 켜면 됩니다.
예: `https://[계정].github.io/weekly-report/team-form.html`

## 4. 사용 흐름

- **팀장**: `team-form.html` 접속 → 팀 선택, 주차(월요일) 확인 → 고객사별 항목 추가하며 입력 → 제출
- **개인**: `individual-form.html` 접속 → 이름/팀 선택 → 이번주/다음주 내용 입력 → 제출
- **부실장님**: `dashboard.html` 접속 → 주차 선택 → 팀별 현황·정체 프로젝트·매출 합계 확인 →
  필요 시 "엑셀 내보내기" 클릭 → 기존 양식과 같은 컬럼 순서의 xlsx 다운로드

## 데이터 구조 (Google Sheets)

Apps Script가 최초 실행 시 아래 두 시트를 자동으로 만듭니다.

**TeamStatus**: `Week | Team | Client | Owner | Project | Stage | LastWeek | ThisWeek | Period | Revenue | SubmittedAt`

**Individual**: `Week | Name | Team | LastWeek | ThisWeek | Notes | SubmittedAt`

`Week`는 매주 시트를 새로 만드는 대신, 그 주 월요일 날짜(`2026-07-06` 형식)로 구분합니다.
과거 주차 데이터도 같은 시트에 계속 쌓이므로, 이후 "주차별 트렌드 비교" 같은 분석도 바로 가능합니다.

## 담당자 클릭 선택 (실제 구성원 스프레드시트 연동)

팀장 폼의 "담당" 자동완성은 별도 시트를 만들 필요 없이, **K-Brain 구성원 현황(HRD사업실/공공AI센터) 스프레드시트**를
직접 참조합니다. (`https://docs.google.com/spreadsheets/d/1zPgN9zMcs5RgWbBc86L1WuPk0sAiTUCoBYxKWEFzgXQ`)

- 그 스프레드시트의 "소속 / 팀명 / 이름" 컬럼을 그대로 읽어옵니다. "사업1팀"처럼 공백 없이 적혀 있어도
  자동으로 "사업 1팀" 표기로 맞춰서 인식합니다.
- 팀명이 비어있는 임원/부실장님(이재우, 김승영 등)은 특정 팀에 속하지 않는 것으로 보고,
  모든 팀에서 담당 후보로 함께 노출됩니다.
- 이 스프레드시트를 계속 최신 상태로 관리해 두시면, DB 스프레드시트 쪽은 아무것도 손댈 필요 없이
  바로 자동완성에 반영됩니다.

⚠️ 이 연동이 동작하려면, Apps Script를 배포/실행하는 Google 계정이 저 구성원 스프레드시트에
**최소 "뷰어" 권한 이상으로 접근 가능**해야 합니다. 접근 권한이 없으면 담당 자동완성만 조용히 비활성화되고
(오류로 전체가 멈추지 않습니다) 다른 기능은 정상 동작합니다.

## 고객사명 / 사업명 자동완성 범위

고객사명·사업명 자동완성과, 클릭 시 담당/단계/사업기간/매출을 함께 채워주는 기능은
**팀 구분 없이 TeamStatus 시트 전체 데이터**를 대상으로 합니다. 즉 다른 팀이 입력한 고객사·사업명도
후보로 뜨고, 같은 고객사+사업명 조합이 있으면 가장 최근 이력 기준으로 자동 채워집니다.

⚠️ `Code.gs`를 최신 버전으로 교체하셨다면, **배포 → 배포 관리 → 수정 → 새 버전 → 배포**를 한 번 더 해주셔야
반영됩니다.


## 기존 양식 데이터 가져오기 (레거시 가져오기)

지금까지 쓰시던 "주간업무 현황" 시트나 "주간보고" 엑셀 데이터를 새 시스템 DB로 가져와서,
새 시스템을 처음 쓰는 주부터 바로 "지난주 내용 불러오기"가 동작하게 할 수 있습니다.

1. **원본 준비**
   - 주간업무 현황 구글시트라면: 가져오고 싶은 주차 탭에서 마우스 우클릭 → **복사 → 다른 스프레드시트로 복사** →
     대상으로 이 DB 스프레드시트(1단계에서 만든 것)를 선택
   - 주간보고 엑셀 파일이라면: 구글 드라이브에 업로드 후 "Google Sheets로 열기"로 변환 → 안의 탭(Sheet1 등)을
     위와 같은 방법으로 DB 스프레드시트에 복사
2. DB 스프레드시트를 열면 방금 복사된 탭이 보입니다. 탭 이름을 확인해 두세요.
3. DB 스프레드시트 상단 메뉴에 **주간업무 도구 → 레거시 시트 가져오기**가 생겼을 거예요.
   (안 보이면 스프레드시트를 새로고침 하세요. Apps Script를 재배포하면 다음 번 스프레드시트를 열 때 메뉴가 생깁니다.)
4. 클릭 후 나오는 입력창에 2번의 탭 이름을 입력 → 확인
5. "가져오기 완료 / 주차: 2026-04-06 / N건 저장됨" 같은 알림이 뜨면 끝입니다.
   주차는 "금주내용 (2026.04.06. ~ ...)" 같은 헤더에서 자동으로 인식합니다.
6. 과거 여러 주차를 가져오고 싶으면 탭별로 2~5번을 반복하시면 됩니다.

가져온 데이터는 TeamStatus 시트에 그대로 쌓이기 때문에, team-form.html의 "지난주 내용 불러오기"가
이 데이터를 인식해서 그대로 이어쓰기를 지원합니다.

⚠️ 이 기능을 쓰려면 `Code.gs`를 최신 버전으로 교체하고 재배포해 주셔야 해요
(배포 → 배포 관리 → 새 버전으로 배포).
